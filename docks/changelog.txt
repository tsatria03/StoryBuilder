New in 1.2.
Fixed a bug where calling a function that does not exist would silently do nothing. An error dialog is now shown with the function name.
Fixed a bug where gotoif only performed string comparisons, causing numeric comparisons to fail. It now supports the same full condition syntax as the if command.
Fixed a bug where calling a function with an invalid target line would push onto the local and call stacks before the check, leaving them permanently unbalanced.
New in 1.1.
Fixed a bug where returning from a function that contained unclosed if blocks would leave leftover entries on the if result stack, causing else and elseif to behave incorrectly in subsequent calls.
Fixed a bug where arrays, inventory, quests, and journal data were being stored in global dictionaries by the command handlers but looked up in separate class member dictionaries by other commands such as spin. This caused the spin command to always find an empty wheel, and could have affected other commands that cross the boundary between class methods and free functions.
Improved the readme documentation for several commands, including return, filter, map, genmap, worldmap, teleport, waitkey, combat, questcheck, questclear, journaladd, journal, and checkpoint. Descriptions now accurately reflect how each command works and what arguments they accept.
Updated the story loader to handle all line ending styles, including Windows CRLF, Unix LF, and old Mac CR. Story files can now be written on any platform.
Fixed a bug in the template story where pressing escape on optional fields would show unresolved variable tokens instead of skipping the follow-up text.
Removed unreachable code after the teleport and worldmap commands.
Fixed a bug where the filter command would match substrings instead of exact values, causing for example filtering for 1 to also match 10 and 11.
Fixed a bug where comparing a variable to an empty string in conditions would always return false.
Fixed a bug where negative numbers were not recognized as numeric values in conditions, causing comparisons like modifier < 0 to always fail.
Fixed a bug where negative modifiers in the dice roller were not being applied correctly due to menu choices being wrapped in quotes.
New in 1.0.
Addressed an issue where returning certain functions with the return command would crash the entire program.
Improved the return command. It can now return actual values such as strings, numbers, and calls to other funcions.
Added the ability to apply aditional settings for controling cancelation types to the writemode command.
Fixed major comparison issues with nested conditions.
Added the ability to configure character restriction settings for typing in input boxes with the new writemode command.
Fixed some bugs with executing various nested loop types.
Added the ability to write nested conditions and loops.
Added the ability to generate various types of passwords with the new genpass command.
Added whildcard support to playing sounds in stories.
Fixed some major issues with managing variables for while loops.
Fixed some major issues with initializing while loops.
Fixed some bugs with interacting in menus.
Fixed some bugs with calling functions
Added the ability to create local variables for writing functions.
Added the ability to create arguments for writing functions.
Added the ability to create functions and while loops in your story.
Fixed an issue where pressing escape in menus and input boxes would not break out of them propperly.
Added the ability to toggle hearing dialog sounds with the new textsound command. You can type this in your stories to disable them at certain points.
Added a large collection of commands to control various asspects of your story. This includes things like dialogs, functions, loops, controls, audio and game systems, variable management, and much much more.
Added a comprehencive readme file to the docks folder that explains the core features of this mini story scripting language.
